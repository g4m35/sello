# First live eBay publish runbook

The deliberate, one-time checklist for Sello's first controlled live eBay
listing. Production publishing is hard-locked behind `EBAY_PRODUCTION_PUBLISH_ENABLED`
and stays OFF until you intentionally run this. Nothing here is automated: a
human enables the flag, reviews the exact payload, and confirms each step.

Never put secrets (keysets, tokens, policy values) in this file or any commit.

## Safety model (how the lock works today)

- `EBAY_PRODUCTION_PUBLISH_ENABLED` defaults to OFF. Only the exact string
  `"true"` enables it (`isEbayProductionPublishEnabled`). Any typo, blank, or
  stray value stays OFF.
- With the flag OFF in production:
  - The UI never renders a live publish button. `publishImplementedFor("ebay")`
    returns `false`, so `canLivePublish` is false on the listing page and the
    publish modal cannot be opened.
  - If the publish route is called anyway, `publishEbayListing` returns a typed
    `EBAY_PUBLISH_NOT_ENABLED` result and makes zero outbound eBay calls; the
    route responds `403` and writes a `publish_blocked` event. No listing is
    created.
  - Readiness and the dry-run preflight still work (they are flag-independent),
    so you can fully prepare a listing without any publish capability.
- The dry-run preflight (`/api/listings/[id]/ebay-preflight`) performs no
  outbound eBay calls of any kind and can never create a listing.

## Preconditions (verify with the flag still OFF)

1. **Environment**: `EBAY_ENV=production` and the production eBay keyset env
   vars are set in Vercel (client id/secret, redirect uri name, token
   encryption key, OAuth state secret). Verify presence, never print values.
2. **eBay account connected**: Settings -> Marketplaces shows eBay connected
   (production OAuth complete, tokens stored encrypted).
3. **Seller config present**: run "Refresh Readiness" in Settings -> Marketplaces
   so payment, fulfillment, and return policy IDs and the merchant location key
   (inventory location) are stored. The final review and publish both require
   all four.
4. **Listing chosen and ready**: the real sneaker listing that shows
   **Ready for eBay**, category **Men's Athletic Shoes / 15709**, **quantity 1**.

## The listing to use

- Item: the sneaker listing currently marked **Ready for eBay**.
- Title: confirmed in the editor (Gemini draft, seller-approved).
- Price: seller-approved price (no invented prices; from the draft).
- Category: **Men's Athletic Shoes / 15709** (inferred high-confidence; a saved
  override always wins).
- Quantity: **1** (explicit resale default, shown in the preflight card).
- Aspects: required item specifics for the category (Brand, Size, Department,
  US Shoe Size, etc.) filled from item data or the seller's saved answers. The
  preflight card lists any still missing, in plain language.

## Final payload preview (before enabling anything)

1. Open the listing, scroll to **eBay publish readiness**, click
   **Check eBay readiness**. Confirm the badge says **Ready for eBay**.
2. Expand **Technical preview** and read the exact `inventoryItem` and `offer`
   payloads that would be sent: SKU, condition, title, price, categoryId,
   availableQuantity, listing policies (payment/fulfillment/return), and
   `merchantLocationKey` (inventory location).
3. This preview is generated by the same builders the real publish uses, so the
   payload you see is the payload eBay would receive.

## Duplicate protection (already enforced)

`assertEbayPublishNotDuplicate` refuses to create a second listing/offer when:

- a prior publish attempt is `QUEUED`, `RUNNING`, or `SUCCEEDED`, or
- the marketplace listing already has an `externalListingId`, an
  `externalOfferId`, or status `LISTED`.

The attempt also carries an idempotency key `"{itemId}:ebay:production"`. A
pre-API readiness failure (no external IDs yet) is the only case that may be
retried.

## Publish attempt / event logging (audit trail)

Every attempt is persisted before any outbound call and updated with the true
result. Expect these rows after a real run:

- `PublishAttempt`: one row, `RUNNING` -> `SUCCEEDED` (or `FAILED`).
- `MarketplaceEvent` (in order on success): `publish_started`,
  `ebay_inventory_item_created`, `ebay_offer_created`, `ebay_offer_published`.
- On failure: `publish_failed` with the failing step. On a flag-off call:
  `publish_blocked`. Nothing is ever recorded as published unless eBay returned
  a listing id.

## Run the first live publish

1. **Enable the flag**: in Vercel, set `EBAY_PRODUCTION_PUBLISH_ENABLED=true`
   for Production, then redeploy so the running build picks it up.
2. Reload the listing. The **Publish** button now appears (gated on
   readiness).
3. Click **Publish**. The modal loads the dry-run preflight and shows the
   **final review**: marketplace (eBay Production), title, price, category,
   quantity, condition, payment/fulfillment/return policies, and inventory
   location.
4. Tick **"I understand this creates a live eBay listing on eBay (production)."**
   The **Create live eBay listing** button only enables once the review is
   ready and the box is checked.
5. Click **Create live eBay listing**. Watch the operations panel: it should
   show the stored **SKU**, **Offer ID**, and **Listing ID**, with status
   **Published**.
6. Verify on eBay (Seller Hub) that the live listing matches the preview.

## Delist recovery path

- In the listing's **Publish operations** panel, **End eBay listing** appears
  only once the channel is `published` with both an offer id and listing id.
- It requires the explicit confirmation "This ends the live eBay listing", then
  calls `/api/listings/delist` with `confirmLiveDelist: true`.
- The handler withdraws the offer (`ebay_offer_withdrawn` event) and moves the
  listing `LISTED` -> `DELISTING` -> `DELISTED`. A failure records
  `delist_failed` and restores `LISTED` with the error, never a fake success.

## Rollback / cleanup checklist (do this right after the test)

1. **Turn the flag OFF**: set `EBAY_PRODUCTION_PUBLISH_ENABLED=false` (or remove
   it) in Vercel Production and redeploy. Confirm the Publish button is gone.
2. **Delist through Sello** if the publish succeeded: use **End eBay listing**
   in the operations panel and confirm.
3. **Confirm status is delisted**: the channel/listing status shows `DELISTED`
   and the operations panel no longer offers **End eBay listing**.
4. **Confirm on eBay**: the listing is ended/withdrawn in Seller Hub.
5. Record the outcome (attempt id, listing id, result) in `HANDOFF.md`. No
   secrets.

## Flag enable / disable quick reference

- Enable: `EBAY_PRODUCTION_PUBLISH_ENABLED=true` in Vercel Production + redeploy.
- Disable: set to `false` or remove the var in Vercel Production + redeploy.
- Only the exact string `true` enables live publishing; everything else is OFF.
